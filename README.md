# MetaComb_git

MetaComb is a drug-combination response prediction project built on top of MAML.
It combines drug molecular graph features with cell (or patient) transcriptome features for binary response prediction.

## Project Structure

- `code/`
  - Main experiment scripts used in this repo.
- `data/`
  - Drug SMILES data, expression matrices, and labeled combination datasets.

## Key Scripts in `code/`

- `code/cell_adapt.py`
  - Core MetaComb MAML pipeline.
  - Source-domain meta-training, then target-domain support/query adaptation and testing.
  - Uses `learn2learn` MAML (`meta_lr=0.0008`, `fast_lr=0.08` in script).
  - Produces checkpoints, split records, per-run/per-seed predictions, and metric summaries.

- `code/cell_general.py`
  - Non-meta baseline workflow (pretrain on source, finetune on target, then test).
  - Reuses target data splits generated by the adapt pipeline.

- `code/5fold_new_zscore_by_train.py`
  - Repeated 5-fold CV on source-domain cell-line data.
  - Configured as 10 repeats x 5 folds with early stopping.
  - Saves fold outputs and aggregated ROC/PR statistics.


## Data Files (Observed in `data/`)

- `4130_drug_smiles_cid.csv`:
  - 4130 drugs with CID/SMILES mapping.

- `cl_label_data_onlysmiles_1016.pickle`:

- `cl_label_data_onlysmiles_bliss_0519.pickle`:

- `log2_tpm_rich_zscore.csv`:
  - Source-domain (data-rich) expression matrix.

- `log2_tpm_poor_zscorebyrich.csv`:
  - Target-domain (data-poor) expression matrix.

- `log2_tpm_patient_zscorebyrich.csv`:
  - Patient expression matrix (3 patients).

- `PDSP_synergy.csv`:
  - Small synergy dataset (20 rows).

- `dict_cell.npy`:
  - Cell mapping object (numpy object array).

## Model Design (Shared Across Scripts)

- Drug encoder (`DrugEncoder`):
  - Converts SMILES to graph.
  - Uses atom/bond features + Morgan fingerprint (64-dim).
  - Applies 3-layer GCN + pooling + MLP fusion.

- Cell encoder (`CellEncoder`):
  - Multi-layer MLP that compresses ~14890 gene features.

- Fusion head (`COMBFC2`):
  - Concatenates drug1, drug2, and cell embeddings.
  - Outputs response probability with sigmoid.

## Task and Evaluation Logic

- Common domain split rule in scripts:
  - Cell lines with sample count > 50 -> source domain.
  - Cell lines with sample count 10 to 50 -> target domain.

- Metrics:
  - AUROC
  - AUPR

- Typical setup:
  - Multi-seed repeats
  - Support/query split for adaptation
  - Early stopping with patience

## Important Reproducibility Notes

- Many scripts contain hard-coded absolute paths (for example `/data`).
  Update path constants before local runs.

- Main dependencies:
  - `torch`, `torch_geometric`
  - `learn2learn`
  - `rdkit`
  - `numpy`, `pandas`, `scikit-learn`, `matplotlib`

- `cell_general.py` scripts depend on split artifacts generated by the adapt pipeline.

## One-Sentence Summary

This repository implements a MAML-based multi-modal predictor for drug-combination response,
with source-to-target adaptation, patient few-shot transfer, and baseline/control pipelines for comparison.
